## 如何使用这些配置

### 1. AI 助手会自动读取这些配置
当你向 AI 提问时，AI 会自动参考这些配置文件，理解项目背景和规范。

### 2. 配置文件会持续更新
随着项目的发展，这些配置文件会不断更新和完善。

### 3. 可以手动指定配置
如果需要 AI 关注特定的配置，可以在提问时明确指出。

## 配置原则

1. **准确性**: 配置内容与实际代码保持一致
2. **完整性**: 覆盖项目的所有重要方面
3. **实用性**: 提供实用的指导和示例
4. **可维护性**: 易于更新和维护
5. **可扩展性**: 方便添加新的配置

## 更新配置

当项目发生重大变化时，需要更新相应的配置文件：

1. **新增功能**: 更新 architecture.md 和相关规则
2. **技术栈变更**: 更新 tech_stack.md
3. **规范调整**: 更新 coding_standards.md
4. **常见问题**: 更新 faq.md
5. **性能优化**: 更新 performance_optimization.md

## 贡献指南

如果你发现配置文件有误或需要补充：

1. 提出 Issue 描述问题
2. 提交 PR 修改配置
3. 在 PR 中说明修改原因

## 注意事项

1. 配置文件使用 Markdown 格式
2. 代码示例使用 TypeScript
3. 保持语言简洁清晰
4. 提供具体的代码示例
5. 注重实用性
